Selection Sort Overview

Selection Sort is a comparison-based sorting algorithm. It sorts an array by repeatedly selecting the smallest (or largest) element from the unsorted portion and swapping it with the first unsorted element. This process continues until the entire array is sorted.

How it Works:

First, the smallest element is found and swapped with the first element. This ensures the smallest element is at its correct position. The process is repeated for the second smallest element and so on, until the entire array is sorted.

Selection Sort Algorithm

                    
    // C++ program to implement Selection Sort
    #include 
    using namespace std;
    
    void selectionSort(vector &arr) {
        int n = arr.size();
    
        for (int i = 0; i < n - 1; ++i) {
            // Assume the current position holds
            // the minimum element
            int min_idx = i;
    
            // Iterate through the unsorted portion
            // to find the actual minimum
            for (int j = i + 1; j < n; ++j) {
                if (arr[j] < arr[min_idx]) {
                    // Update min_idx if a smaller
                    // element is found
                    min_idx = j;
                }
            }
    
            // Move minimum element to its
            // correct position
            swap(arr[i], arr[min_idx]);
        }
    }
    
    void printArray(vector &arr) {
        for (int &val : arr) {
            cout << val << " ";
        }
        cout << endl;
    }
    
    int main() {
        vector arr = {64, 25, 12, 22, 11};
    
        cout << "Original array: ";
        printArray(arr); 
    
        selectionSort(arr);
    
        cout << "Sorted array: ";
        printArray(arr);
    
        return 0;
    }

Complexity Analysis of Selection Sort

Time Complexity: O(n2)

  • One loop to select an element of the array one by one: O(n).
  • Another loop to compare the selected element with every other array element: O(n).
  • Total Complexity = O(n) * O(n) = O(n2).

Auxiliary Space: O(1) (in-place sorting).

Advantages of Selection Sort

  • Simple and easy to understand, making it ideal for teaching sorting concepts.
  • Requires only constant O(1) extra memory space.
  • Fewer memory writes compared to other algorithms, which is beneficial in scenarios where write operations are costly.

Disadvantages of Selection Sort

  • Has a time complexity of O(n2), making it slower than algorithms like Quick Sort or Merge Sort.
  • Not stable, meaning it does not maintain the relative order of equal elements.

Applications of Selection Sort

  • Useful for teaching fundamental sorting mechanisms and algorithm design.
  • Suitable for small lists where the overhead of more complex algorithms isnâ€™t justified.
  • Ideal for systems with limited memory due to its in-place sorting capability.
  • Used in simple embedded systems where simplicity and resource constraints are key factors.
